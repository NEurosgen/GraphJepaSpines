# Graph Jepa Spines

Правила ведения веток

Ветка main  - содержит работающий код для обучения модель с помощью гидра с логированием и изменением параметров через конфиг. То есть главный файл train_model должен имено обучать модель потом в cli добавиться файл val/test model Для итого отслеживания работы модели

Ветка exp  - использует код модели для проведения разного рода экспериментов 
Так как у нее нет обязаности именно обучать конкрентный эксземпляр модели а ветка нужна для провееняи экспериментов , то 
train_model в ней заменен на run_experiment 
В файле run_experimtnt описана логика проведения эксперимента и основные компоненты
Разные по замыслу эксперименты проводятся в разных ветках (их не нужно мержить для удобства к возвращению эксперимента)

В файле Readme Эксперимента должен быть конспект замысла или же отчет по проведенному эксперименту
